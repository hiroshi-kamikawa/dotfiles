# フェーズ2：オブジェクト抽出

## ゴール

フェーズ1のタスク一覧から **操作対象＝オブジェクト（名詞）** を抽出し、整理・統合する。
ここで決まる主オブジェクトが、後のナビゲーションとビューの骨格になる。

## 進め方

1. `01-usecases.vN.md`（最新版）を読む。
2. 各タスクから名詞を抜き出す。
3. **主オブジェクトか属性かを判定する**。判定の目安：
   - ユーザーがその「集合（一覧）」を見たい／個々を選んで操作したい → **主オブジェクト**。
   - 他のオブジェクトに付随する値で、それ単体を一覧・操作しない → **属性**（そのオブジェクトのプロパティ候補）。
   - 例：「案件」は主オブジェクト。「案件の金額」「ステータス」は案件の属性。
4. 同義語・表記ゆれを **統合** する（例：「取引先」と「顧客」が同一なら片方に寄せる）。
5. 各オブジェクトの **アクション候補** を、由来タスクの動詞から拾う。
6. オブジェクト間の関連の気配（「案件は見積を持つ」等）があればメモしておく（厳密化は次フェーズ）。

## チェック

- どのオブジェクトにも紐づかない動詞が残っていないか。残るならオブジェクトの抽出漏れを疑う（`references/ooui-principles.md` 原則5）。
- 主オブジェクトが多すぎる／少なすぎる場合は粒度を見直す。

## 出力フォーマット

`docs/ooui/<app-name>/02-objects.vN.md`

```markdown
# オブジェクト抽出 v1

対象アプリ: <app-name>

## 主オブジェクト

| オブジェクト | 説明 | 由来タスク | 主要アクション候補 |
|--------------|------|------------|--------------------|
| 案件 | 商談の単位 | T01,T02 | 登録／進捗更新／受注化 |
| 見積 | 案件に対する金額提示 | T03 | 作成／承認／差し戻し |
| 顧客 | 取引先（「取引先」と統合） | … | 登録／編集 |

## 属性候補（参考）
- 案件: 金額, ステージ, 担当者, 期日 …
- 見積: 金額, 明細, 承認状態 …

## 関連の気配（次フェーズで厳密化）
- 案件 1 — * 見積
- 顧客 1 — * 案件
```

## 完了時

`references/critique-and-iteration.md` の完了プロトコルに従う。
- 提示する可視化: 主オブジェクト一覧＋統合した同義語
- 特に見る批評観点: オブジェクトの粒度、主オブジェクト／属性の判定の妥当性、どこにも属さない動詞の有無（抽出漏れ）
- 次フェーズ: 概念モデル（ただしユーザーが満足するまで進めない）
