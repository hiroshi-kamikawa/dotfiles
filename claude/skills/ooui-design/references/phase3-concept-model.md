# フェーズ3：概念モデル

## ゴール

各オブジェクトの **プロパティ・アクション・関連（多重度）** を確定し、概念モデルとして可視化する。
可視化は Mermaid の classDiagram を使う。

## 進め方

1. `02-objects.vN.md`（最新版）を読む。
2. 各主オブジェクトについて確定する：
   - **プロパティ**：そのオブジェクトが持つ属性（名前・型の粒度でよい。DB設計ではないので過剰に詳細化しない）。
   - **アクション**：そのオブジェクトに対してユーザーが行う操作（フェーズ1の動詞を吸収）。
   - **関連と多重度**：他オブジェクトとの関係を 1対1 / 1対多 / 多対多 で表す。
3. どのオブジェクトにも属さないアクションが無いか最終確認する。
4. 関連の向き（どちらから辿るのが自然か）も意識しておく（フェーズ4で使う）。

## 可視化（Mermaid classDiagram）

- class＝オブジェクト、属性＝プロパティ、メソッド＝アクション。
- 関連は多重度つきで書く。

```markdown
## 概念モデル図

\`\`\`mermaid
classDiagram
  class 顧客 {
    名称
    業種
    +登録する()
    +編集する()
  }
  class 案件 {
    タイトル
    ステージ
    金額
    +登録する()
    +進捗更新する()
    +受注化する()
  }
  class 見積 {
    金額
    承認状態
    +作成する()
    +承認する()
    +差し戻す()
  }
  顧客 "1" --> "*" 案件 : 持つ
  案件 "1" --> "*" 見積 : 持つ
\`\`\`
```

（実際の出力では `\`\`\`` のエスケープは不要。通常の mermaid コードブロックとして書く。）

## 出力フォーマット

`docs/ooui/<app-name>/03-concept-model.vN.md`

```markdown
# 概念モデル v1

対象アプリ: <app-name>

## 概念モデル図

- 関連: 顧客(多対1), 見積(1対多)

- …
```
