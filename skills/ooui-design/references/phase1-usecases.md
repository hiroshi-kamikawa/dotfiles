# フェーズ1：ユースケース抽出

## ゴール

このアプリで **ユーザーがやりたいこと** を、「対象 ＋ 動詞」のタスクとして洗い出す。
UI（画面）ではなく、業務上の行為のレベルで列挙する。

## 進め方（ロール起点 × 業務フロー起点のハイブリッド）

質問は一度に詰め込まず、対話的に。次の順で掘る。

1. **目的の確認**：このアプリは何のためのものか、一言で確認する（未確定なら聞く）。
2. **ロール起点**：誰が使うかを洗い出す（例：営業、管理者、経理、外部顧客）。各ロールについて「このアプリで何をしたいか」を引き出す。
3. **業務フロー起点**：主要な業務の流れを時系列で辿る（例：問い合わせ→案件化→見積→受注→請求）。各ステップで発生する行為を拾う。
4. **取りこぼし確認**：例外・差し戻し・参照だけの行為（閲覧、検索、集計）も拾う。CRUD の観点（作る・見る・変える・消す）で抜けを点検する。

各タスクは粒度を揃える。「画面を開く」ではなく業務行為（「案件を登録する」「見積を承認する」）で書く。

## 抽出のコツ

- 「〜する」の形で書く。動詞と対象（名詞）が必ず含まれるようにする。
- 1タスク1行。複合的なものは分解する。
- このとき出てくる **名詞** は次フェーズのオブジェクト候補なので、控えておく。
- 細かいUIや画面遷移にはまだ立ち入らない。

## 出力フォーマット

`docs/ooui/<app-name>/01-usecases.vN.md` に次の形式で書く。

```markdown
# ユースケース（タスク）一覧 v1

対象アプリ: <app-name>
目的: <一言>

## ロール
- <ロールA>: <説明>
- <ロールB>: <説明>

## タスク一覧

| ID | ロール | タスク（対象＋動詞） | 補足 | 名詞候補 |
|----|--------|----------------------|------|----------|
| T01 | 営業 | 案件を登録する | | 案件 |
| T02 | 営業 | 案件の進捗を更新する | ステージを進める | 案件 |
| T03 | 管理者 | 見積を承認する | 金額しきい値あり | 見積 |
| … | | | | |
```

## 完了時

`references/critique-and-iteration.md` の完了プロトコル（書き出し → index更新 → 提示 → 専門批評 → 深掘り質問 → 反復ゲート）に従う。
- 提示する可視化: タスク一覧の要点＋抽出した名詞候補
- 特に見る批評観点: タスクの粒度が揃っているか、CRUD・例外・参照行為の抜け、各動詞が後でオブジェクトに吸収できるか
- 次フェーズ: オブジェクト抽出（ただしユーザーが満足するまで進めない）
