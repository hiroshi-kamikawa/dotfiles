# HK Skill Creator

## このスキルについて

CodexとClaudeの両方で利用できる、個人用スキルを作成・更新するためのスキルです。スキル名、保存場所、メタデータ、記述言語などの共通ルールを定め、管理スクリプトによる雛形作成と検証を提供します。

エージェントが実行時に読む指示は英語の `SKILL.md` に置き、このREADMEは人間がスキルを理解・レビューするための日本語資料として扱います。

通常のスキルでは補助的なREADMEを置かないことが推奨されますが、このdotfilesリポジトリでは、本人が後から設計判断を確認できることを優先し、日本語READMEを意図的なローカル規約として必須にしています。`SKILL.md` からREADMEを参照させず、エージェントの実行に必要な情報はREADMEへ置きません。

## 効果的な使い方

新しいスキルの作成、既存スキルの改修、スキル構成の検証を依頼するときに使います。実現したい作業と、スキルを起動してほしい状況を具体的に伝えると、適切な `description` とワークフローを設計しやすくなります。

作成後は次の観点でレビューすると効果的です。

- 想定した依頼でスキルが選ばれるか
- CodexとClaudeのどちらでも実行可能か
- 繰り返し使う価値のある手順だけが含まれているか
- スクリプトや参照資料が必要最小限か
- このREADMEから用途と実装判断を理解できるか

## 実装の背景と決定

個人用スキルを複数のエージェント環境で共有するため、保存先を `$HOME/dotfiles/skills` に固定し、スキル名には `hk-` プレフィックスを付けます。実行仕様はプラットフォーム固有の機能へ過度に依存させず、CodexとClaudeの双方で解釈できる英語で記述します。

一方、スキルを作った理由や適切な利用方法、実装時の判断は、本人が読み返しやすい日本語で残します。この情報を実行コンテキストへ常に入れる必要はないため、`SKILL.md` ではなく `README.md` に分離しています。通常の実行ではエージェント向け指示を簡潔に保ちつつ、スキルのレビュー時には同じディレクトリ内で背景まで確認できます。

名前の正規化、雛形作成、構造検証には `scripts/manage_skill.py` を使います。意味のあるワークフロー設計や日本語READMEの内容は機械的に生成・判定しきれないため、エージェントによるレビュー工程として残しています。

ローカル検証では、必須ファイル、名前、メタデータ、READMEの必須セクション、OpenAI UIメタデータの基本制約を確認します。その後、環境内で利用できるCodexやClaudeの標準バリデータも実行します。標準バリデータ本体や依存パッケージがない場合は「未実行」として報告し、ローカル検証の成功と混同しません。
