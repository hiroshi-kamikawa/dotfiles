# HK Setup Astro

## このスキルについて

Astroのminimalテンプレートを起点に、Git、Tailwind CSS 4、daisyUI 5、Partytown、Sitemapまでを揃えたスターターを再現するためのスキルです。新規セットアップだけでなく、既存プロジェクトが想定どおりに構成されているかを監査する用途にも対応します。

確認対象は依存パッケージだけではありません。AstroのVite設定、TypeScriptの厳格設定と`@/*`パスエイリアス、Tailwind CSSとdaisyUIのCSSディレクティブ、スタイルシートの読み込み経路、Partytownの配信アセット、SitemapのXMLと本番URL、Gitリポジトリの有無、production buildの成否までを一連のセットアップとして扱います。

## 効果的な使い方

新しい空ディレクトリで再現する場合は、「`$hk-setup-astro` でスターターを作成して」と依頼します。既存プロジェクトを確認する場合は、「`$hk-setup-astro` で現在のセットアップを監査して」のように、変更してよいか、監査だけに留めるかを添えると安全です。

対象ディレクトリが明確でない場合はパスも指定してください。既存プロジェクトでは、使用中のパッケージマネージャーや構成を尊重し、不足箇所だけを直す運用を想定しています。

## 実装の背景と決定

実際に行った「Astro minimal作成、Git初期化、`astro add tailwind`、daisyUI追加、Partytown導入、Sitemap導入、TypeScript設定、CSS設定、型チェックとビルド確認」という流れを、再利用できる手順へ整理しました。空ディレクトリへの足場作成と依存追加は`scripts/bootstrap.sh`、構成監査とビルド成果物の確認は`scripts/astro_starter.py`へ集約しています。毎回同じ確認をエージェントが手作業で繰り返さず、CodexとClaudeのどちらからも同じ判定を利用するためです。

一方、既存ファイルとの競合解消、CSSの読み込み範囲、Calendarコンポーネントを本当に使っているかといった文脈依存の判断は、意図的にスクリプトへ押し込んでいません。スクリプトは安全な初期化と機械的な一次判定を担当し、エージェントが結果と差分を確認して最終判断する構成です。

`tsconfig.json`ではAstroのstrict設定を継承したうえで、未使用のローカル変数と引数をエラーとして検出し、`@/*`を`src/*`へ解決する標準構成を採用しています。既存設定がある場合は、ファイル全体を置き換えず`compilerOptions`へ必要項目をマージします。Astroのminimalテンプレートには共通Layoutがないため、セットアップ時は既存ページからCSSを読み込み、Layoutを作ること自体は必須にしていません。

Partytownはパッケージ追加だけで完了とせず、ビルド後のHTMLに初期化コードが入り、`~partytown`アセットが生成されることまで確認します。実際のサードパーティースクリプトを移す場合は、互換性を確認して`type="text/partytown"`を付け、必要なグローバル関数だけを`forward`します。

SitemapはAstroの`site`設定がなければ正しいURLを生成できないため、`SITE_URL`環境変数を基準にしました。リポジトリから本番ドメインを確認できない場合に架空のドメインを固定せず、ローカルでは`http://localhost:4321`へフォールバックします。本番デプロイ時には実ドメインの設定が必要です。検証ではテスト用URLを渡して、サイトマップインデックスとページURLの両方へ反映されることを確認します。

daisyUI 5.7.0では、未使用のCalendarコンポーネントに含まれるCally向けセレクターがCSS最適化時の警告になる場合がありました。Calendarを使っていないことを確認した場合に限り、daisyUI公式の`exclude: calendar`設定で生成対象から外します。Calendarを利用するプロジェクトや別種の警告では、この回避策を機械的に適用しません。

パッケージの具体的なバージョン番号はスキルへ固定していません。CLIや公式手順は更新されるため、実行時に解決された依存関係とロックファイルを正とします。一方、Tailwind CSS 4のViteプラグイン方式とdaisyUI 5の`@plugin`方式は監査項目として明示し、古いTailwind CSS 3の設定を誤って合格扱いしない設計にしています。

`bootstrap.sh`は空でない対象を拒否し、既存プロジェクトを誤って上書きしません。`astro_starter.py`はPython標準ライブラリだけで動作し、`audit`と`verify`を提供します。`verify --run-build`は安全なテスト用`SITE_URL`を渡してビルドし、Sitemap XMLのURLとPartytownのHTML・配信アセットをまとめて確認します。
